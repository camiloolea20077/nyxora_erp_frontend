# docs/ui — Diseño de interfaz (skill interface-design)

Aquí van los artefactos de diseño producidos con la skill **interface-design** antes de codificar:
bocetos/layout, jerarquía visual, estados (cargando/vacío/error), responsive y decisiones de UX por módulo.

Convención: un archivo por módulo o pantalla, p. ej. `productos.md`, `pos.md`, `dashboard.md`.
El objetivo es **consistencia profesional** entre módulos: reusar patrones (header de módulo, tabla card,
dialogs) definidos en `.claude/rules/02-ui-primeng-tailwind.md`.
